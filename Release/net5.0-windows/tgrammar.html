TGrammar: программа, которая по заданной грамматике генерирует все цепочки языка в некотором диапазоне длин

Использование программы

Программа позволяет пользователю вводить терминальные, нетерминальные символы контекстно-свободной грамматики, правила грамматики и начальный (целевой) символ, после чего рекурсивно выполняет подстановку всех возможных правил грамматики для получения слов из заданного количества (диапазона) выводимых символов-терминалов.

Пользовательский интерфейс

Программа имеет меню «Действия» и набор управляющих элементов.

Меню «Действия» позволяет запросить справку, краткую информацию о программе и выйти из программы.

Набор управляющих элементов состоит из групп «Грамматика», «Задать встроенную грамматику» и группы запуска и получения результата.

1) «Грамматика»

Группа позволяет пользователю сформировать контекстно-свободную грамматику в форме Бэкуса-Наура. Терминальные символы последовательно (по одному) вводятся в левое текстовое поле, при нажатии клавиши ввода или прилежащей кнопки со стрелкой соответствующий символ размещается в списке терминалов. В качестве терминалов допустимы буквы алфавита в нижнем регистре, скобки, знаки препинания и т.п.

Нетерминальные символы вводятся схожим образом в центральное текстовое поле, при нажатии клавиши ввода или прилежащей кнопки со стрелкой соответствующий символ размещаются в списке нетерминалов. Допустимы буквы алфавита в верхнем регистре.

Множество правил вида α→β вводится в правой части группы. В малое текстовое окно вводится нетерминал, в правое – соответствующее ему правило замены. Допустимо ввести пустое правило: для этого при вводе нетерминала в левое текстовое поле достаточно оставить правое текстовое поле пустым (по умолчанию оно будет заполнено символом ε, обозначающим пустую цепочку).

Под списками терминалов, нетерминалов и правил расположены:

строка ввода начального (целевого) нетерминального символа;

управляющие элементы для определения длины результирующей цепочки;

переключатель типа вывода (лево- или правосторонний);

строка, формирующая форму Бэкуса-Наура для заданной грамматики.

2) «Задать встроенную грамматику»

Позволяет задать одну из нескольких встроенных грамматик. Для этого в выпадающем списке нужно выбрать нужную грамматику и нажать кнопку «Применить». Обращаем внимание на то, что содержимое раздела «Грамматика» будет переписано без предупреждения!

3) Раздел ввода-вывода содержит список вывода и управляющий элемент «Дерево». При нажатии кнопки «Запуск» заданная грамматика считывается программным модулем и по ней формируются все возможные цепочки. После формирования готовые цепочки загружаются в список вывода. При нажатии на какой-либо элемент списка производится загрузка дерева вывода в соответствующий управляющий элемент. При нажатии кнопки «Сброс» программа возвращается к исходным значениям.

Поскольку программа моделирует комбинацию всех допустимых сочетаний всех правил, время ее работы может быть значительным.

Для работы программы необходимо задать нужную контекстно-свободную грамматику, выбрать тип вывода и диапазон длины слова (строки нетерминалов) и нажать «Запуск».

Известные ошибки

Программа потенциально допускает ввод «экзотических» символов (как минимум в список терминалов): в случае невозможности обработки программа выдаст ошибку.
